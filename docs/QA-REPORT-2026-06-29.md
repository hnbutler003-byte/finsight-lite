## 3. T001: Silent-failure Audit

Examined every in-scope DB write: `deleteClass`, `removeEnrollment`, `createLessonPlan`, `createLessonQuizQuestion`, `upsertLeaderboardSnapshot`, and the `IStorage.getClassProgressSummary` interface vs implementation mismatch.

### Confirmed failures

**ISSUE-01: `upsertLeaderboardSnapshot` (server/supabase.ts:489) -- FIXED**
- Behaviour before fix: on Supabase write error, logged + called `captureError` but returned `void`. The caller at `students.ts:1542` caught nothing and returned `{ok:true}` to the client even when the leaderboard write failed.
- Fix applied: added `throw new Error(...)` inside the error block. The existing `catch (e: any)` at students.ts:1549 now returns `400 {message}` instead of a false positive `200`.

**ISSUE-02: `IStorage.getClassProgressSummary` interface missing `opts` parameter (server/storage.ts:145) -- FIXED**
- Behaviour before fix: the TypeScript interface declared `getClassProgressSummary(classId: number)` but the implementation at storage.ts:942 accepted `(classId, opts?)` and the call site at students.ts:1134 passed `{ limit, offset }`. The mismatch made the interface incorrect and would have caused type errors on any strict caller.
- Fix applied: updated interface to `getClassProgressSummary(classId: number, opts?: { limit?: number; offset?: number }): Promise<any>`.

### Confirmed NOT silent

- `deleteClass` -- throws with message on DB failure.
- `removeEnrollment` -- throws "Not enrolled" when not found.
- `createLessonPlan` -- Supabase path throws `[Supabase] ...` prefix; route catch returns 500.
- `createLessonQuizQuestion` -- same pattern.

### No additional HOLDs

Both issues were unambiguous (no design decision required). No items held.

## 7. T005: Fixes Applied

### Fix 1 -- `upsertLeaderboardSnapshot` now throws on error

**File:** `server/supabase.ts` line 502-506
**Change:** Added `throw new Error(...)` inside the Supabase error block after `captureError`. The throw propagates to the existing `catch` block in the calling route and returns a clear `400 {message}` to the client.

Before:
```typescript
if (error) {
  console.error("[Supabase] upsertLeaderboard:", error.message);
  captureError(...);
  // silently returned void -- caller got {ok:true}
}
```

After:
```typescript
if (error) {
  console.error("[Supabase] upsertLeaderboard:", error.message);
  captureError(...);
  throw new Error(`[Supabase] upsertLeaderboard failed: ${error.message}`);
}
```

### Fix 2 -- `IStorage.getClassProgressSummary` interface aligned with implementation

**File:** `server/storage.ts` line 148
**Change:** Added `opts?: { limit?: number; offset?: number }` parameter to match the implementation signature and call site.

Before: `getClassProgressSummary(classId: number): Promise<any>;`
After: `getClassProgressSummary(classId: number, opts?: { limit?: number; offset?: number }): Promise<any>;`

### Typecheck result

`npx tsc --noEmit` exits 0 after both fixes. No new type errors introduced.
