TASK 1 — Org admin temporary class teacher reassignment

This adds the ability for an org admin to temporarily reassign a class to a different teacher (for substitute coverage, a teacher on leave, etc.), without building full multi-teacher/co-teaching support. This is intentionally a simpler solution than the previously-assessed class_teachers junction table approach, since classes currently have a single teacher_id field.

STAGE 2 — Build the reassignment feature

1. On whichever org admin page shows the organization's classes, add a "Reassign Teacher" action per class. This should open a simple selector showing the other teachers already in the organization (from the existing /org/teachers list), and let the org admin pick a new teacher for that class.
2. On reassignment, update the class's teacher_id to the newly selected teacher. The class's existing students, lesson history, content, and data must remain completely intact, only the teacher_id changes.
3. The previous teacher should immediately lose access to that specific class (since access is enforced via teacher_id matching), and the newly assigned teacher should immediately gain full access (view roster, upload content, give feedback, etc.) as if they had always been that class's teacher.
4. Log this reassignment with a timestamp and which org admin performed it (a simple audit trail, consistent with the existing deletion_log pattern already used elsewhere in the app), so there's a record of who changed a class's teacher and when.
5. Show the org admin a brief confirmation after reassignment (e.g. "Class reassigned to [Teacher Name]"), and optionally show the current teacher's name clearly on each class row so it's always obvious who's currently responsible for it.
6. This is NOT permanent co-teaching, only one teacher is ever assigned to a class at a time. Make this single-teacher-at-a-time behavior clear in the UI so it isn't confused with simultaneous shared access.
