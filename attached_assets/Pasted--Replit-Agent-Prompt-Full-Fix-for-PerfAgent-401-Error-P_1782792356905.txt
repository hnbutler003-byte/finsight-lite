## Context

A diagnostic pass already confirmed the following, do not re-diagnose,
go straight to fixing:

- The error is `401 invalid x-api-key` from PerfAgent
  (`server/perfAgent.ts`), a code-health scanner that calls Claude on a
  schedule.
- All four AI-calling files (`perfAgent.ts`, `investmentExplainer.ts`,
  `routes/ai.ts`, `routes/orgs.ts`) read the same two Replit-managed
  integration variables: `AI_INTEGRATIONS_ANTHROPIC_API_KEY` and
  `AI_INTEGRATIONS_ANTHROPIC_BASE_URL`. There is no bare
  `ANTHROPIC_API_KEY` anywhere, do not introduce one, do not generate a
  new key from console.anthropic.com, that would bypass Replit's
  billing gateway and not fix anything.

  (`const anthropic = new Anthropic({...})` at the top of the file),
  unlike the other three files which build their client lazily, fresh
  on each request. If the integration variables aren't populated yet at
  the exact moment the server boots and this module is first imported,
  the client freezes with a bad/empty key permanently, even if the
  variables become available a moment later.
