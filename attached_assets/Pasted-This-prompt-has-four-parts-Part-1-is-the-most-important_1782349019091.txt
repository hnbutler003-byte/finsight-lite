PART 1 — Systemic fix for card contrast across all money games (root cause, not patches)

The recurring problem: individual game cards have been built with hardcoded colors at some point (an inline style or a hardcoded hex/Tailwind color with no dark: variant), causing text or backgrounds to become illegible depending on whether the user is in light or dark mode. This has already happened with the BISX/JSE ticker symbols earlier this session and is now showing up across multiple money games. Patching individual instances has not solved the underlying pattern, since new instances keep appearing.

STAGE 1 — Audit every game for the root cause

For each of the 7 money games, search the actual component code for:
1. Any inline style={{ color: ... }} or style={{ backgroundColor: ... }} usage.
2. Any hardcoded Tailwind color class (e.g. text-gray-800, bg-white, text-black) used WITHOUT a corresponding dark: variant.
3. Any card, panel, or score/points display that does not use the existing glass-card, glass-card-coral, glass-card-teal, or other semantic component classes already defined in index.css.

Report every instance found, file and line, organized by game, before fixing anything.

STAGE 2 — Fix every instance found, using only semantic tokens

For every instance found in Stage 1:
1. Replace inline hardcoded color styles with the existing semantic Tailwind classes already defined (text-foreground, text-card-foreground, text-muted-foreground, bg-card, etc.) or the existing glass-card component classes, so colors automatically adapt to light and dark mode rather than being fixed values.
2. Any score, points, stock holdings, or currency amount displayed inside a game card must use text-foreground or text-card-foreground (or the existing amber/secondary token for money-related figures, consistent with established color usage rules), never a hardcoded color.
3. Any card background must use bg-card, glass-card, or another existing semantic surface class, never a hardcoded white or dark hex value.

STAGE 3 — Add a lasting safeguard against this recurring

Add a short, clearly commented section near the top of index.css (as a code comment, extending the existing Dark Mode Regression Checklist already added earlier this session) specifically calling out: "Game card components are a recurring source of hardcoded-color bugs.
